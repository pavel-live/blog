## Правила контента и SEO для блога

Этот файл помогает агентам (особенно аналитику, планировщику и разработчику) генерировать и проверять фичи с учётом требований к контенту.

### 1. Общие принципы

- **Цель текста** — информировать и показать экспертизу, а не «продавать любой ценой».
- **Стиль** — спокойный, профессиональный, без пустых эпитетов.
- **Длина материалов**:
  - Короткие анонсы / новости: N–M слов.
  - Проектные кейсы: от X слов, с понятной структурой.

Уточни реальные числа под свой блог.

### 2. Структура страницы проекта / записи

Рекомендуемый шаблон для нового проекта (аналог `content/02.en/01.projects/.../project.txt`):

```text
title: Название проекта
-
date: Год Месяц
-
client: Клиент, Город, Страна.
-
material: Основные материалы / технологии.
-
awards:

Год **Название награды**, Место.

-
preview-ratio: 2x1 | 2x2 | 3x2 (соотношение превью)
-
is-dark: yes | no
-
summary:

Краткое описание проекта (2–4 предложения).

-
content:

Основной текст кейса / статьи, с абзацами и, при необходимости, цитатами.
-
```

### 3. Требования к SEO

- **Заголовки**:
  - Уникальные и осмысленные `title`.
  - H1 = основное название проекта / статьи.
- **Метаданные**:
  - `description` с 1–2 предложениями, содержащими ключевые слова, но без переспама.
- **URL**:
  - Читаемые slugs: латиница, дефисы вместо пробелов.
  - Без технических параметров в постоянных ссылках.

### 4. Изображения и медиа

- Для каждого проекта:
  - Минимум одно основное изображение высокого качества.
  - Указание формата и соотношения сторон превью.
- Обязательно:
  - `alt`-тексты, описывающие изображение по смыслу.

### 5. Как использовать этот файл агентами

- Аналитик / планировщик:
  - учитывают формат `project.txt` и требования к структуре;
  - предлагают только такие изменения, которые не ломают контент-модель.
- Разработчик:
  - проверяет, что новые поля контента вписываются в существующий формат;
  - добавляет проверки и тесты на наличие ключевых полей.
