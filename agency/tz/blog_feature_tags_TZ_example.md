## ТЗ: Страница тегов для блога

### 1. Общее описание

Реализовать страницу тега, которая позволяет пользователю просматривать все материалы (проекты и записи блога), отмеченные выбранным тегом. Страница должна быть доступна по человекочитаемому URL и соответствовать общему стилю блога.

### 2. Юзер-кейсы

#### UC-01: Просмотр материалов по тегу

- **Актёры**:
  - Посетитель сайта.
  - Система блога.

- **Предусловия**:
  - В системе есть хотя бы один тег.
  - К тегу привязаны материалы (проекты или записи блога).

- **Основной сценарий**:
  1. Пользователь кликает на тег на странице проекта или записи.
  2. Система перенаправляет пользователя на страницу тега `/tag/{slug}/`.
  3. Система отображает название тега.
  4. Система отображает список связанных материалов с краткими описаниями.
  5. Система отображает навигацию по страницам (если материалов больше N).

- **Альтернативные сценарии**:
  - **А1: Для тега нет материалов**:
    1. Система отображает сообщение «Материалов с этим тегом пока нет».
  - **А2: Неверный slug тега**:
    1. Система отображает страницу 404 или переадресует на список тегов (уточнить).

- **Постусловия**:
  - Пользователь видит релевантные материалы по тегу или понятное сообщение об их отсутствии.

- **Критерии приёмки**:
  - ✅ Страница доступна по URL `/tag/{slug}/`.
  - ✅ При клике на тег в материале пользователь попадает на страницу этого тега.
  - ✅ Список материалов отсортирован по дате (от новых к старым).
  - ✅ При отсутствии материалов выводится понятное сообщение.

### 3. Нефункциональные требования

- Страница должна быть доступна для индексации поисковыми системами.
- Время генерации страницы — в разумных пределах (до X мс, уточнить).

### 4. Открытые вопросы

1. Нужна ли отдельная страница со списком всех тегов?
2. Нужна ли фильтрация по типу материалов (проекты / блог)?
3. Как именно обрабатывать неверный slug: 404 или редирект?
