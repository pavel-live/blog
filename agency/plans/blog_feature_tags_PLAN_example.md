## План реализации фичи: Страница тегов для блога

### 1. Общее описание

Фича `blog_feature_tags` реализуется в рамках существующего блога. Требуется добавить маршрут и шаблон страницы тега, интегрировать его с текущими страницами проектов и записей блога.

### 2. Зависимости

- Утверждённое ТЗ: `agency/tz/blog_feature_tags_TZ_example.md`
- Описание проекта: `agency/PROJECT_OVERVIEW.md`
- Архитектура блога: `agency/ARCHITECTURE.md`

### 3. Задачи

#### Задача 1.1: Определить модель тегов и связи с материалами

- Проверить, как сейчас хранятся теги и связи «материал ↔ тег».
- При необходимости — предложить минимальные изменения модели.
- Обновить документацию в `ARCHITECTURE.md` (подраздел «Модель тегов»).

#### Задача 1.2: Добавить маршрут и контроллер/обработчик страницы тега

- Добавить маршрут `/tag/{slug}/`.
- Реализовать обработчик, который:
  - находит тег по `slug`;
  - загружает связанные материалы;
  - передаёт данные в шаблон.

#### Задача 1.3: Реализовать шаблон страницы тега

- Создать шаблон с:
  - заголовком = название тега;
  - кратким описанием (если есть);
  - списком материалов с краткой информацией.
- Учесть требования из `CONTENT_RULES.md` (SEO, структура страницы).

#### Задача 1.4: Добавить ссылки на теги на страницах материалов

- На страницах проектов и записей блога:
  - отобразить список тегов;
  - сделать их кликабельными, ведущими на `/tag/{slug}/`.

#### Задача 1.5: Тестирование и документация

- Добавить тесты (юнит/интеграционные) на:
  - корректную работу маршрута;
  - отображение материалов по тегу;
  - обработку пустого списка материалов.
- Обновить документацию:
  - добавить раздел про страницу тегов в `PROJECT_OVERVIEW.md` или отдельный файл;
  - при необходимости — отдельный файл `agency/architecture/tags_ARCH_example.md`.

### 4. Приоритеты

- Начать с задач 1.1–1.2, затем 1.3–1.4, завершить 1.5.
