## Архитектура блога (черновик)

Опиши здесь текущую архитектуру блога так, чтобы её мог понять агент-архитектор и планировщик.

### 1. Технологический стек

- **Язык / платформа**:  
  Пример: PHP + собственный движок / статический генератор / Node.js / Python.
- **Шаблонизатор / рендеринг**:
  - Серверный рендеринг / статический рендер / SPA.
- **Хранение контента**:
  - Структура папки `content/` (языки, рубрики, проекты).

### 2. Структура репозитория

Заполни реальные пути для этого проекта:

- `content/` — исходные тексты и метаданные.
- `app/` или эквивалент — код рендера и парсеры (например, `app/parsers/...`).
- `public/` / `static/` — статические ресурсы.
- `config/` — конфигурация сайта, роутинг, локали.

### 3. Роутинг и URL

- Как формируются URL для:
  - страниц проектов;
  - записей блога;
  - тегов/категорий;
  - поиска (если есть).
- Какие ограничения по URL (языковой префикс, слэш в конце, форматы дат и т.п.).

### 4. Сборка и деплой

- Как запускается сборка / генерация:
  - Команда: `...`
  - Где указывать окружение (prod/stage).
- Где хостится сайт:
  - Пример: GitHub Pages / VPS / другой хостинг.

### 5. Нефункциональные требования

- **Производительность**:
  - Цели по времени загрузки страниц.
- **SEO**:
  - Требования к структуре HTML, метатегам, JSON-LD (если есть).
- **Доступность**:
  - Базовые требования (alt-тексты, контраст, доступность навигации).

### 6. Как использовать этот файл агентами

- Архитектор опирается на него при предложении изменений.
- Планировщик использует реальные пути к файлам и модулям.
- Разработчик сверяет с ним места внесения изменений.
