## 01_orchestrator (оркестратор мультиагентной разработки)

Полная версия промпта: `../agents/01_orchestrator.md`

Оркестратор координирует работу всех агентов: аналитика, ревьюеров, архитектора, планировщика, разработчика. Он отвечает за переходы между этапами, лимиты итераций review и остановку процесса при блокирующих вопросах.

### Как применять в блоге

Используй оркестратора, когда хочешь пройти **полный цикл** по крупной фиче:

1. Подготовь:
   - описание задачи в `agency/tasks/...`;
   - актуальные `PROJECT_OVERVIEW.md`, `ARCHITECTURE.md`, `CONTENT_RULES.md`.
2. Запусти оркестратора с этими ссылками во входных данных.

### Пример запуска (концептуально)

```bash
cursor-agent -f --model auto \
  -p "$(cat ../agents/01_orchestrator.md)" \
  -p "Постановка задачи: см. agency/tasks/blog_feature_tags_example.md. Описание проекта: см. agency/PROJECT_OVERVIEW.md. Архитектура: см. agency/ARCHITECTURE.md."
```

Оркестратор далее будет формировать запросы к другим агентам (аналитику, архитектору, планировщику и т.п.), опираясь на структуру, описанную в исходном промпте.
