## Задача: Страница тегов для блога

### 1. Контекст

В блоге есть статьи и проектные страницы, которым назначаются теги. Сейчас пользователь не может перейти на страницу тега и посмотреть все материалы с этим тегом.

### 2. Что нужно сделать

Реализовать страницу тега, доступную по URL вида `/tag/{slug}/`, которая:

- показывала название тега и краткое описание (если есть);
- выводила список связанных материалов (проекты, записи блога);
- поддерживала пагинацию для длинных списков;
- учитывала язык (если есть мультиязычность).

### 3. Цель

- Упростить навигацию по темам.
- Повысить глубину просмотра и время на сайте.

### 4. Ограничения

- Не ломать существующую структуру URL.
- Использовать существующую модель хранения тегов, если она уже есть.

### 5. Ожидаемый результат

- Страница тега доступна и индексируется поисковиками.
- На странице выводятся релевантные материалы с корректной сортировкой (по дате публикации).
