## Agency: мультиагентная разработка для блога

Эта папка содержит инфраструктуру для работы с мультиагентной схемой разработки (аналитик → архитектор → планировщик → разработчик → ревьюеры) применительно к этому блогу.

### Структура

- `PROJECT_OVERVIEW.md` — краткое описание блога и его целей.
- `ARCHITECTURE.md` — обзор технической архитектуры блога.
- `CONTENT_RULES.md` — правила контента: SEO, структура страниц, метаданные.
- `status.md` — текущий статус работы агентов по активным задачам.
- `open_questions.md` — открытые вопросы, требующие ответа человека.

Подпапки:

- `tasks/` — исходные постановки задач для блога (что нужно сделать и зачем).
- `tz/` — технические задания, созданные аналитиком.
- `architecture/` — архитектурные документы по результатам работы архитектора.
- `plans/` — общие планы работ по фичам.
- `task_descriptions/` — детальные описания задач для разработчика (task_X_Y.md).
- `test_reports/` — отчёты о тестировании по задачам.
- `prompts/` — промпты агентов, скопированные из репозитория `agents` и адаптированные под блог.

### Типовой цикл для новой фичи блога

1. Создай файл постановки задачи, например:  
   `agency/tasks/blog_feature_tags.md` — опиши, какую фичу хочешь (страница тегов, поиск, рекомендации и т.п.).
2. Запусти **аналитика** (`prompts/02_analyst_prompt.md`), передав:
   - ссылку на файл задачи из `tasks/`,
   - `PROJECT_OVERVIEW.md` и `CONTENT_RULES.md`,
   - при необходимости — фрагменты кода.
   Аналитик создаст ТЗ в `tz/` и список блокирующих вопросов.
3. После утверждения ТЗ запусти **планировщика** (`prompts/06_agent_planner.md`), передав:
   - утверждённое ТЗ из `tz/`,
   - `ARCHITECTURE.md`,
   - ссылки на нужные файлы кода.
   Планировщик создаст план в `plans/` и задачи в `task_descriptions/`.
4. Для каждой задачи используй **разработчика** (`prompts/08_agent_developer.md`), передавай:
   - нужный `task_X_Y.md` из `task_descriptions/`,
   - ссылки на файлы проекта,
   - `CONTENT_RULES.md`, если важно SEO/контент.
   Разработчик предложит изменения в коде и создаст отчёт о тестах в `test_reports/`.
5. Проверяй изменения через **ревьюера кода** (`prompts/09_agent_code_reviewer.md`) и отмечай прогресс в `status.md`.

Все файлы в `agency/` можно коммитить в репозиторий блога — это документация и артефакты работы агентов.
